Настройка NFS на Proxmox:
Установка NFS-сервера на одном из хостов:
На одном из серверов (например, Proxmox-хосте или отдельной виртуальной машине) установите NFS-сервер:
sudo apt update
sudo apt install nfs-kernel-server

Создание общей директории:
Создайте директорию, которая будет использоваться как общее хранилище:
sudo mkdir -p /mnt/nfs-share

Настройка экспорта директории:
Добавьте запись в файл /etc/exports, чтобы указать, что эта 
директория будет доступна по сети:
sudo nano /etc/exports

Добавьте строку:
/mnt/nfs-share 192.168.100.0/24(rw,sync,no_subtree_check)
Здесь 192.168.100.0/24 — это подсеть, в которой находятся ваши виртуальные машины. Замените на нужную вам сеть.

Применение настроек:
Примените изменения и запустите NFS-сервер:
sudo exportfs -a
sudo systemctl restart nfs-kernel-server

Подключение NFS на виртуальных машинах
На каждой виртуальной машине, которая должна иметь доступ к общему хранилищу:
Установите NFS-клиент:
sudo apt update
sudo apt install nfs-common

Смонтируйте NFS-хранилище:
sudo mkdir -p /mnt/nfs-share
sudo mount 192.168.100.1:/mnt/nfs-share /mnt/nfs-share
Здесь 192.168.100.1 — IP-адрес сервера, где запущен NFS. Замените его на актуальный адрес.

Чтобы автоматически монтировать директорию при загрузке, добавьте запись в /etc/fstab:
192.168.100.1:/mnt/nfs-share /mnt/nfs-share nfs defaults 0 0
